REST

About

  • Stands for: Representational State Transfer

  • Used for sending data bidirectionally between the browser and the server.

  • Commonly the browser uses AJAX to make requests to the server.

Swagger

Swagger is a powerful open source framework backed by a large ecosystem of tools that helps you design, build, document, and consume your RESTful APIs.

http://swagger.io/

  • Helps you to plan and organize your REST API.

  • Documents your API for other's to consume.

  • Has tools that enforce your API.

  • Specification - How to write swagger documents.

  • Swagger UI - A tool for writing swagger documents.

Exercise

  1. Create a person API. A person object will look like this:

     {
     id: 0,
     name: 'Bob',
     age: 20
 }

  2. We'll put the routes in ./server/routes/people.js and we'll use the express router middleware.

  3. Set up a POST for /people that can add a person, not needing the ID because that will be determined by the server.

    • Use body-parser middleware for JSON to decode body from the browser.

    • Store the added person in memory. (Later we'll use databases, but for now we'll use memory.)

  4. Set up a GET for /people that get's an array of people.

  5. Set up a GET for '/people/:personId' that get's a single person by ID.

  6. Create a PUT for '/people/:personId' that is used to update a person.

  7. Create a DELETE for '/people/:personId' that is used to delete a person.
